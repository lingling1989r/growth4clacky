## 四、竞品标签体系（用于对比营销）

### 4.1 直接竞品

#### OpenClaw（龙虾）
| 标签维度 | 标签内容 |
|------|------|
| **产品状态** | 本地优先、个人助手、未达企业级 |
| **安全标签** | 🔴 CVE-2026-25593、密码暴力破解、4万+公网暴露实例 |
| **安全事件** | 中毒事件、Claude源码泄露、Moltbook数据泄露 |
| **用户评价** | "也是壳"、"Bug太多"、"一卡就完" |
| **使用门槛** | 权限过大、配置复杂 |
| **企业遭遇** | 三星韩国禁止、NVDB预警、微软定性"不可信代码执行环境" |
| **核心弱点** | 安全系统性风险、权限失控、提示注入无解 |

#### Hermes（爱马仕龙虾）
| 标签维度 | 标签内容 |
|------|------|
| **产品状态** | 太早期、无自恢复、一卡就完 |
| **接入问题** | 阿里云百炼不支持、文档缺失 |
| **稳定性** | "一卡就完了" |
| **核心弱点** | 稳定性极差、API兼容差 |

#### Code agent（Cursor/Windsurf等）
| 标签维度 | 标签内容 |
|------|------|
| **Token消耗** | 首轮20000+ tokens，成本高 |
| **幻觉问题** | "解决了bug但引入新bug" |
| **软技能** | 不通知、不推送、要人盯着 |
| **多开困境** | 项目代码变多后左边开始幻觉 |
| **核心弱点** | 硬技能在线、软技能缺失 |

### 4.2 间接竞品

| 产品 | 核心标签 | 可攻击弱点 |
|------|---------|---------|
| Devin | 全自动黑箱 | 不透明、不可控 |
| Nanobot | 港大/4000行/Python | 生态不足/学术圈 |
| NanoClaw | 容器级安全 | 功能最受限/Anthropic绑定 |
| Copilot | L2辅助/IDE插件 | 非Agent架构/单文件 |

## 五、用户画像标签体系（流量入口）

### 5.1 核心目标人群

#### 人群A：独立开发者 / 个人开发者
**标签**：独立开发者、一个人公司、一万美金、出海开发者、小白学编程

**痛点标签**：
- 零经验想build但不懂代码
- 需要完整产品（不是demo）
- 不想学命令
- 希望AI代劳重复性工作

**命中内容方向**：
- "零基础用AI编程做出完整产品"
- "不需要懂命令，对话就能开发"
- 一周上线产品案例

#### 人群B：技术创业者 / CTO / 技术负责人
**标签**：连续创业者、技术背景、融资、创业公司

**痛点标签**：
- 研发成本高（千万→百万）
- 团队扩张慢
- 希望一个人能顶团队用

**命中内容方向**：
- "研发成本从千万降到百万"
- "CTO用Clacky一个人做完整项目"
- 创始人故事

#### 人群C：有代码基础但被AI编程困住的人
**标签**：Cursor用户、Copilot用户、每天被抽卡折磨、技术人

**痛点标签**：
- 软技能不爽（不通知、不推送）
- Token消耗太快
- 多开对比
- "雇了个牛逼实习生但不来报告"

**命中内容方向**：
- "Code agent首轮20000 tokens，Clacky只需5000"
- "你用的agent是不是也会解决一个问题引来三个新bug"
- 傲雪的多开对比案例

#### 人群D：移动办公 / 远程办公人群
**标签**：ToDesk用户、移动办公、远程开发者、数字游民

**痛点标签**：
- 背着电脑出去担心没电
- 远程看不到进度
- 需要ToDesk配合使用

**命中内容方向**：
- "地铁上继续写代码"
- "任务跑完自动推送，手机看一下就行了"
- 不需要ToDesk的远程开发

#### 人群E：出海创业者
**标签**：出海、中国开发者全球化、万美元市场

**痛点标签**：
- 不知道海外市场付费意愿
- 本地化/多语言需求

**命中内容方向**：
- "一万亿美金全球市场"
- "海外付费意愿远超国内"
- 出海案例
