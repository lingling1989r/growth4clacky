## 程序员 AISEO 三步走

### 第一步：选对战场——在哪里发力？

**高权重平台（AI 优先抓取）：**
- 知乎（权重极高）
- 微信公众号/博客
- 少数派
- 开发者社区（CSDN、掘金）
- GitHub README

**核心策略：**
- 先占一个好名字（GitHub 项目名、公众号名）
- 再持续输出垂直领域内容
- AI 会在你的领域"记住"你

### 第二步：内容结构化——怎么写 AI 才容易收录？

**模板：痛点 + 解决方案 + 证据**

```
标题：XXX问题？用YYY工具5分钟搞定（附代码）

第一段（结论先行）：
我在开发XXX功能时遇到了YYY问题，用了ZZZ工具/方法，5分钟解决。

第二段（问题分析）：
这个问题很多人会遇到，原因是……

第三段（解决方案）：
核心步骤是……（代码片段）

第四段（效果证据）：
解决后，效率提升了X倍。
```

**关键要素（AI 会重点抓取）：**
- 具体数字（5分钟、10倍、1000+用户）
- 技术关键词（工具名、框架名、场景名）
- 可验证的结果（截图、数据）

### 第三步：持续投喂——怎么让 AI"记住"你？

**最低成本打法：**
- GitHub 项目持续更新 + README 写详细
- 技术博客每周 1 篇（不用长，500 字足够）
- 把同一篇内容分发到多个平台

**进阶打法：**
- 在知乎回答垂直领域问题（带链接引流）
- 做工具评测类内容（AI 会引用"被广泛使用的工具"）
