## 四、核心功能与产品体系

### 4.1 三大核心 Agent

| Agent | 功能 | 说明 |
|-------|------|------|
| **Chat Agent** | 对话式编程辅助 | 自然语言交互，即时答疑 |
| **Git Agent** | Git 操作自动化 | 自动处理 commit、PR、代码管理 |
| **Task Plan Agent** | 任务规划与拆解 | 自主规划长程任务，分配子任务 |

### 4.2 核心功能亮点

#### ① Task Time Machine（任务时间机器）
- 代码快照技术，实时追踪、审计、调整 AI 的每一次变更
- 增强开发者对开发过程的可见性和控制

#### ② Collaborative Multitasking（协作多任务）
- 支持多线程协作，在一个 Workspace 里异步协同
- 团队成员可并行推进多个开发任务，快速迭代产品

#### ③ 智能环境构建
- **自动搭建开发环境**：用户导入项目后，Clacky 会第一时间自动搭建开发环境（这是全球范围内少有的功能）
- 内置 MySQL、Postgres、Redis 等中间件
- 支持 web 单体类项目在线开发与调试（golang、Python、Node、Vue/React、Ruby 等数十种框架及语言）

#### ④ 项目化思维
- 在给出需求列表时，引导用户先为任务给出项目名称，明确目标与边界
- 同步提示任务规模、消耗积分
- 每个关键步骤前等待用户确认，执行后可随时撤回

#### ⑤ AI 自主规划与自我排查
- 强大的 AI 自主规划能力
- 自动拆分需求并分配子任务
- 内置自我排查机制，降低 debug 成本

### 4.3 产品哲学

> **"你思考架构，AI 完成代码。"**

- AI 为主，人类为辅（L3 定位）
- 在每个关键步骤前支持用户确认
- 执行后可随时撤回（不是黑箱，是白盒）
- 约等于初级工程师水平，部分场景达中高级水平，未来持续进化

## 十一、创始人通话记录补充（2026年1月，与罗傲雪的深度对话）

### 产品路线：CLI 工具 + 移动端

**新产品动态**：Clacky 团队正在开发一个 CLI 工具（内部名称未完全确定），定位是 OpenClaw/Code 的轻量平替，但整合 Clacky 的核心优势。

**Token 效率对比**：
- Clacky CLI 首轮启动仅约 **5000 tokens**
- Code agent 首轮就约 **20000+ tokens**（"它 token 还没开始干就已经 2 万个 token 了"）
- 缓存命中率 99%，缓存后 $0.008/请求，未命中 $0.02

**两种确认模式**：
- **confirm save 模式**：安全操作不询问（如当前目录文件操作），文件写入类才询问
- **auto approve 模式**：连文件写入也不问，大量工作自动完成

**Agent 能力对比（傲雪亲测）**：
- 傲雪同时开着 Code agent 和 Clacky 两边，用同一个 base model（MiniMax 中转）
- Code agent："查 bug 的时候它确实会把这个问题解决了，但是它会引起其他问题"，项目代码变多后容易左边出现幻觉问题
- Clacky：更稳定，适合贯穿整个项目生命周期（项目初期构建 + 后期维护）
- "我中间有事中间有一次我烧的太多了，code 给烧没了"

### 核心用户痛点（傲雪作为种子用户分享）

**AI 编程的软技能缺失**：
- "你雇了一个实习生，他虽然很牛逼，技术很牛逼，但是他办完一个事，他就在那里傻等，然后他也不来告诉你，需要你来去确认"
- "你希望的不只是一个工具，你还是想要一个类人的 agent，希望它软技能上比较友好"
- "硬技能现在慢慢可以打满格，但软技能上面超级不爽"

**远程开发的三大痛点**：
1. **无推送**：AI 干完活不知道，必须远程连进去看（ToDesk/向日葵）
2. **需反复确认**：中间卡住了、让你点 yes/no，也不知道，需要主动去确认
3. **背着电脑跑**：背着电脑出去担心没电

### 移动办公产品规划（老李透露的未来方向）

> "我有个画面：我在想要不要搞这个云端的方案，把装在云上，然后提供 session 通知。画面上是点一下就换成手机去控制，再点一下就是云端去开发。"

- **Session 推送**：任务完成自动推送通知到手机
