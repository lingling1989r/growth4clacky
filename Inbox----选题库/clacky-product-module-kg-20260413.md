# Agent 产品模块 KG — 从产品架构做知识点下钻

> **方法论**：把 Agent 产品拆解成「功能模块」，每个模块再下钻到用户关心的「维度」和「具体参数」
> **用途**：产品对比选型、内容营销知识点覆盖、竞品攻击坐标

---

## 总览图

```
Agent 产品
├── 1. 大脑（LLM Engine）
├── 2. 记忆系统（Memory）
├── 3. 工具能力（Tool System）
├── 4. 安全与权限（Security & Permission）
├── 5. 执行环境（Runtime Environment）
├── 6. 用户界面（Interface）
├── 7. 协作与团队（Collaboration）
├── 8. 成本与计费（Cost & Billing）
├── 9. 稳定性与可靠（Reliability）
└── 10. 生态与扩展（Ecosystem）
```

## 模块 1：大脑（LLM Engine）

### L1：基础模型层

| L2维度 | L3具体参数 | 用户痛点/关心点 | 竞品表现 |
|--------|-----------|----------------|---------|
| **模型提供商** | OpenAI GPT-4 / Anthropic Claude / Google Gemini / Meta Llama / 国产大模型 | 被"卡脖子"担忧/合规/成本 | OpenClaw主推OpenAI，国产模型支持差 |
| **模型版本** | GPT-4o / Claude 3.5 Sonnet / Claude 3.7 / GPT-4.5 | 版本迭代快，跟不上就落后 | - |
| **上下文窗口** | 128K / 200K / 1M / 200K | 上下文窗口太小，项目大了就崩（抽卡） | Code agent 200K崩，Claude 3.5 200K也崩 |
| **推理速度** | TTFT / TPS / Latency | "AI太慢"打断心流体验 | Step 3.5解决OpenClaw速度问题 |
| **多模型切换** | 支持3种 / 5种 / 10种+ | 不想绑定单一供应商 | Hermes仅支持自研，Clacky支持多模型 |
| **国产模型适配** | 百度/阿里/腾讯/字节 | 企业合规要求/数据不出海 | NanoClaw/Hermes对国产模型适配差 |

### L1：Token 效率层（用户最关心的成本指标）

| L2维度 | L3具体参数 | 用户痛点/关心点 | 竞品表现 |
|--------|-----------|----------------|---------|
| **首轮 Token 消耗** | 5000 / 10000 / 20000+ | 首轮消耗大 = 每次启动都烧钱 | Code agent 20000+ tokens vs Clacky 5000 |
| **缓存命中率** | 70% / 90% / 99% | 缓存高 = 同样任务便宜 | Clacky宣传99%缓存 |
| **缓存节省策略** | 无 / 基础 / 智能 | 缓存用不用差3-5倍成本 | - |
| **上下文复用** | session内 / 跨session | 切项目要重新喂context | - |
| **输出 Token 效率** | 平均输出长度 / 压缩率 | 废话多 = 浪费Token | - |
